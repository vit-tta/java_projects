# Financial Manager Console Application
## Описание проекта
Finance - консольное Spring Boot приложение, которое позволяет вести учет личных финансов. Система предоставляет полное управление финансовыми данными (работа с категориями, операциями и счетами) с автоматическим пересчетом балансов и поддержкой различных форматов данных.
## Принципы SOLID:

1. **S - Single Responsibility Principle (Принцип единственной ответственности):**
    - "BankAccount" отвечает за счета
    - "Category" отвечает за категории
    - "Operation" отвечае за операции
    - "MenuMain" отвечает за создание и вывод основного меню
    - "_Menu" отвечают за разные пункты меню
    - "ScannerInputSystem" обеспечивает ввод с клавиатуры
    - "_Exporter/Importer" отвечает за экспорт/импорт файлов 
    - и тд...
2. **O - Open/Closed Principle (Принцип открытости/закрытости):**
    - Есть возможность добавить/расширить систему без изменений существующего кода:
        - Благодаря интерфейсам можно добавлять новый формат экспорта
3. **L - Liskov Substitution Principle (Принцип подстановки Лисков):**
    - Объекты в программе заменяеются на экземпляры их подтипов без изменения правильности программы
4. **I - Interface Segregation Principle (Принцип разделения интерфейсов):**
    - Представлено несколько разных интерфейсов (BankAccountStorage, CategoryStorage, OperationStorage)
5. **D - Dependency Inversion Principle (Принцип инверсии зависимостей):**
    - все классы реализуют соответствующие интерфейсы, что показывает зависимость от абстракции, а не от конкретной реализации
    - зависимости передаются через конструкторы
    - используется DI-Контейнер (Spring) для управления зависимостями
## Принципы Grasp:
1. **High Cohesion (Высокая связность):**
    - OperationService содержит всю логику работы с операциями и балансом
    - MenuMain отвечает только за навигацию по меню
    - Каждый экспортер знает только о своем формате
2. **Low Coupling (Низкая связанность):**
    - Модули связаны через интерфейсы
    - Изменения в одной части системы (например, в логике хранения) не затрагивают другие части
3. **Information Expert:**
    - Хранилища обрабатывают данные, которые они хранят, сервисы инкапсулируют бизнес-логику своих доменных областей
4. **Creator:**
    - Объекты создаются в подходящих местах (Сервисы создают доменные объекты, фабрики экспорта создают соответствующие экспортеры)
## Архитектурные решения:
1. **DI-Контейнер:** Используется Spring для управления зависимостями, что упрощает процесс управления компонентами
2. Поддержка как **in-memory**, так и файлового хранения данных
3. **Расширяемая система** импорта/экспорта с поддержкой новых форматов
4. **Наследование и полиморфизм:** Используется для того, чтобы избежать дублирования кода, структурировать объекты и иметь возможность расширения без изменения существующего кода
5. **Инкапсуляция:** Все поля классов являются привытными
## Паттерны GoF:
1. **Builder**
   - Используется для создания доменных объектов BankAccount, Category и Operation. Паттерн упрощает создание объектов со множеством параметров
2. **Delegate**
    -  Меню делегирует задачи ввода данных InputSystem. Это разделяет ответственность между представлением и обработкой ввода
3. **Decorator**
    - ScannerInputSystem адаптирует стандартный Scanner Java к потребностям приложения, добавляя валидацию и обработку ошибок
4. **Facade**
    - Сервисы и FileManager предоставляют упрощенные интерфейсы к сложным системам. Фасад скрывает сложность внутренних взаимодействий, предоставляя клиентам простые методы
5. **Template Method**
    - DataExport и DataImport определяют общий алгоритм экспорта/импорта, оставляя реализацию специфических шагов подклассам. Устраняет дублирование кода в классах
6. **Strategy**
    - FileManager использует стратегии для выбора алгоритмов экспорта и импорта. Это позволяет динамически выбирать формат данных во время выполнения и легко добавлять новые форматы
7. **Proxy**
    - File-хранилища действуют как прокси для InMemory-хранилищ, добавляя функциональность сохранения в файлы без изменения основной логики хранения
8. **Singleton через DI контейнер**
    - Spring DI контейнер обеспечивает единственный экземпляр сервисов и хранилищ в рамках приложения
## Основные функции:
- Полное управление банковскими счетами, категориями, операциями: создание, просмотр всех, редактирование, удаление
- Управление категориями операций с разделением на доходы и расходы
- Управление финансовыми операциями с автоматическим обновлением балансов счетов
- Проверка на остаток средств на балансе
- Экспорт данных в файл Csv(Json)
## Инструкция по запуску:
* Сборка проекта: mvn clean compile
* Запуск проекта: mvn spring-boot:run
* Запуск тестов: mvn test
