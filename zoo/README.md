# Zoo Manager Console Application
## Описание проекта
Zoo manager - консольное приложение, которое позволяет вести учет животных и инвентаря в Московском зоопарке. Принимать их на баланс, выводить отчет по количеству животных, потребляемом
ими килограммов еды в день, формировать списки тех животных, которые могут быть
помещены в контактный зоопарк. Выводить наименование и инвентаризационные номера
вещей, животных, стоящих на балансе зоопарка.
## Принципы SOLID, примененные в проекте:

1. **S - Single Responsibility Principle (Принцип единственной ответственности):**
    - "Animal" и его подклассы отвечают за животных 
    - "Thing" и его подклассы отвечают за вещи
    - "VeterenaryClinic" отвечает за здоровье животных
    - "_Manager" для управления операциями с животными или вещами (два разных класса)
    - "_Repository" для сбора информации о сущностях (два отдельных класса)
    - "Menu" отвечает за создание и вывод меню 
    - "_AddForMenu" отвечают за разные пункты меню 
    - "ScannerInputSystem" обеспечивает ввод 
2. **O - Open/Closed Principle (Принцип открытости/закрытости):**
    - Есть возможность добавить/расширить систему без изменений существующего кода:
      - Можно добавить новые виды животных или вещей, путем создания соответствующих подклассов в родительские классы(Animal, Thing)
      - Благодаря интерфейсам можно добавлять новые реализации хранения данных и способ ввода данных
3. **L - Liskov Substitution Principle (Принцип подстановки Лисков):**
    - Объекты в программе заменяеются на экземпляры их подтипов без изменения правильности программы
4. **I - Interface Segregation Principle (Принцип разделения интерфейсов):**
    - Представлено несколько разных интерфейсов (IAlive, IInventory, InputSystem, AnimalRipository, InventoryRepository)
5. **D - Dependency Inversion Principle (Принцип инверсии зависимостей):**
    - все классы реализуют соответствующие интерфейсы, что показывает зависимость от абстракции, а не от конкретной реализации
    - зависимости передаются через конструкторы
    - используется DI-Контейнер (Spring) для управления зависимостями

## Архитектурные решения:
1. **DI-Контейнер:** Используется Spring для управления зависимостями, что упрощает процесс тестирования и управления компонентами
2. **Паттерн Builder:** Используется в меню для обеспечения безопастного конструирования объекта(в слачае где в конструкторе больше 2 параметров) 
3. **Паттерн Repository:** Обеспеченна абстракция между бизнес-логикой и слоем данных, а также присутствует возможность изменения реализации хранения
3. **Наследование и полиморфизм:** Используется для того, чтобы избежать дублирования кода, структурировать объекты и иметь возможность расширения без изменения существующего кода
4. **Инкапсуляция:** Все поля классов являются привытными 
5. **Unit-тесты**
## Основные функции:
- Возможность добавления новых животных разных типов (Wolf, Tiger, Rabbit, Monkey) в зоопарк
- Проверка его состояния здоровья (здоров /не здоров)
- Возможность информировать о том какое количество килограмм еды нужно всем животным(можно получить с помощью отчета)
- Возможность учитывать уровень доброты и в зависимости от этого предоставлять информацию о тех животных, которые могут быть отправлены в контактный зоопарк
## Инструкция по запуску:
* Сборка проекта: mvn clean compile
* Запуск проекта: mvn spring-boot:run
* Запуск тестов: mvn test
